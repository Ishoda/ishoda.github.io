Habit Tracker & Wellness App - HappyMe

A personal wellness Android application designed to encourage healthy routines through habit tracking, mood monitoring, hydration management and personal progress insights.

Project Overview

Maintaining healthy habits often requires using multiple applications to track routines, hydration, and emotional well-being. HappyMe brings these essential wellness activities together into a single mobile application that provides a seamless and personalized user experience.
The application enables users to manage daily habits, record moods, monitor water intake, and review their overall progress while maintaining all personal data locally on the device. By combining a clean interface with motivational content and personalized insights, HappyMe helps users stay focused on long-term lifestyle improvements.

Problem Statement

Many people struggle to maintain healthy routines because their wellness activities are scattered across different applications or not tracked at all. This makes it difficult to stay consistent and understand personal progress over time.
HappyMe solves this problem by providing an all-in-one wellness platform where users can manage habits, monitor emotional well-being, track hydration and review daily progress from a single application.

Solution

HappyMe integrates habit management, mood tracking, hydration monitoring and progress reporting into one centralized Android application. The app provides personalized greetings, motivational quotes, visual progress indicators, reminder notifications and detailed reports, creating an engaging experience that encourages users to develop healthier daily routines.

Key Features

Home Dashboard

The home screen provides an overview of the user's daily wellness activities, including:

  • Daily habit checklist
  • Motivational quotes
  • Water intake progress
  • Seven-day mood summary chart
  • Personalized greeting
  • Completed habits automatically removed from the active list to improve focus

Routine Management

Users can manage their daily routines through a dedicated habit management system.
Features include:

  • Add new habits
  • Edit existing habits
  • Delete habits
  • Mark habits as completed
  • Habit history
  • PDF progress report generation
  • Personalized onboarding for first-time users

Completed habits are automatically moved below pending habits to improve usability.

Mind Tracking

The Mind section focuses on emotional well-being by allowing users to:

  • Record daily moods
  • Write personal reflections
  • Review mood history
  • Generate PDF mood reports
  • Visualize emotional patterns over time

Fuel Tracker

The Fuel page encourages healthy hydration habits through:

  • Interactive eight-cup water tracker
  • Add or remove water intake
  • Daily hydration goal monitoring
  • Reminder notifications
  • Automatic reset at midnight

MeTime Profile

The profile section allows users to:

  • View completed habit statistics
  • Track daily streaks
  • Manage preferences
  • Switch between Light and Dark Mode
  • View history
  • Log out securely

Application Architecture

HappyMe follows a Single Activity Architecture with multiple Fragments, allowing each feature to remain modular while providing a consistent user experien

    MainActivity
  • Home Fragment
  • Routine Fragment
  • Mind Fragment
  • Fuel Fragment
  • MeTime Fragment
The application follows the MVVM (Model–View–ViewModel) architectural pattern with a Repository Layer, ensuring a clear separation between the user interface, business logic and data management.

Data Management

HappyMe combines Room Database and SharedPreferences for efficient local data persistence.

Room Database

    Stores structured application data including:
  • User profiles
  • Habits
  • Habit completion history
  • Mood entries
  • Hydration records
    • Room entities are connected through relationships and foreign keys, while Kotlin Coroutines perform database operations on background threads to maintain smooth application performance.
    Kotlin Flow provides real-time updates between the database and the user interface.

SharedPreferences

Used to store lightweight configuration data, including:

  • Theme preferences
  • User settings
  • Reminder preferences
  • Last selected options
  • Basic application configuration

CRUD Operations

HappyMe implements complete CRUD functionality to manage user data including habits, mood entries, hydration records, and preferences.

Create

Allows users to create new user profiles, habits, mood entries, and hydration records. New data is stored securely using the Room Database.

Read

Retrieves user-specific information such as daily habits, completion history, mood records, and hydration progress. Kotlin Flow provides real-time data updates.

Update

Enables modification of habits, completion status, mood details, hydration levels, and user preferences.

Delete

Removes unwanted habits, outdated records, and user data while maintaining proper database management.
