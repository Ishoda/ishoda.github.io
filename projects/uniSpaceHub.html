UniSpace Hub - Smart Campus Operations Platform

A full-stack university management platform developed to digitize facility booking, maintenance requests, and campus operational workflows.

Project Overview

UniSpace Hub is a full-stack Smart Campus Operations Platform developed to digitize university facility management and maintenance workflows.
The system provides a centralized platform for managing campus resources, facility booking requests, maintenance incidents, ticket tracking and operational communication. The application follows a modern full-stack architecture with a React-based frontend, Spring Boot REST API backend and cloud-hosted database.
The project was focusing on REST API development, frontend-backend integration, database management, security practices and scalable software architecture.

Business Problem

Traditional university facility management processes often depend on manual booking systems, paper-based maintenance reporting, and disconnected communication channels.
These approaches create challenges in:

  • Tracking facility availability
  • Managing booking requests
  • Monitoring maintenance issues
  • Assigning technicians
  • Tracking service progress
  • Maintaining operational visibility
UniSpace Hub addresses these challenges by providing a centralized digital platform for facility operations and maintenance management.

System Architecture

The system follows a modern three-layer architecture consisting of a React frontend, Spring Boot REST API backend, and cloud-hosted database.

Frontend Layer

React.js application responsible for user interaction, routing, form validation, dashboards, and API communication.

Backend Layer

Spring Boot REST API handles business logic, authentication, validation, ticket workflows, and database operations.

Data Layer

Cloud-hosted database used for storing facilities, bookings, tickets, users, comments, and notifications.

Individual Contribution - Maintenance & Incident Ticket Management

Key Implementations:

  • Designed and developed the complete ticket lifecycle: OPEN → IN_PROGRESS → RESOLVED → CLOSED (with REJECTED status).
  • Built RESTful APIs for ticket creation, retrieval, status updates, technician assignment/claiming, comments and attachments.
  • Implemented ticket creation with categories, priorities, descriptions, facility linking, and image evidence uploads.
  • Developed technician workflows including ticket claiming, status management and resolution notes.
  • Implemented ticket comments and attachment management features.
  • Designed and developed a real-time SLA dashboard to help administrators and technicians prioritize maintenance work effectively.

Backend Development

  • Implemented layered architecture (Controller → Service → Repository) Spring Boot and Spring Data JPA.
  • Implemented CRUD operations for tickets, comments, and attachments.
  • Applied validation rules and error handling to ensure reliable API communication.

Frontend Development

  • Built responsive and interactive user interfaces using React.js.
  • Developed ticket creation forms, ticket dashboards, status tracking views, and management pages.
  • Implemented API integration using Axios for frontend-backend communication.
  • Applied component-based architecture to improve code reusability and maintainability.

Cloud & Additional Integrations

  • Integrated Cloudinary for secure cloud-based image storage of ticket attachments.
  • Worked with a cloud-hosted database environment using Aiven.
  • Implemented secure file handling practices for uploaded images.

Key Features

Facility and Asset Catalogue

Manage campus resources including lecture halls, laboratories, meeting rooms, and equipment.

Online Facility Booking Workflow

Users can request facility bookings while administrators review, approve, or reject requests.

Maintenance Incident Reporting System

Users can submit maintenance issues with descriptions, priorities, categories, and image attachments.

Ticket Assignment and Status Tracking

Technicians and administrators can manage tickets and monitor resolution progress.

Role-Based Access Control

Supports different user roles including User, Technician, and Admin.

SLA Dashboard

Monitors ticket service-level performance using time-to-first-response and time-to-resolution metrics to identify delayed or high-risk tickets.

Notification Management

Users receive updates regarding booking approvals, ticket status changes, and comments through the notification system.

REST API Development

  • Designed and implemented RESTful API endpoints following standard HTTP methods (GET, POST, PUT, DELETE).
  • Developed APIs for ticket creation, updates, comments, attachments, and SLA monitoring.
  • Implemented validation and meaningful error responses for reliable client-server communication.
  • Tested API functionality using Postman.
